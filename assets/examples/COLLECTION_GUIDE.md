# 示例图片收集指南

## 📋 需要的示例图片类别

为了全面测试 DeepSeek-OCR 在不同复杂场景下的识别能力，建议收集以下4类示例图片：

---

## 1. 📄 发票类 (Invoices)

### 已包含
- ✅ `vat_invoice.png` - 增值税电子普通发票（复杂表格）

### 建议补充的类型

#### 1.1 增值税专用发票
- **特点**: 包含更多税务信息、发票代码、发票号码、税率等
- **复杂度**: 高
- **用途**: 测试复杂税务信息提取

#### 1.2 普通发票
- **特点**: 标准格式、商品明细、金额计算
- **复杂度**: 中
- **用途**: 测试标准发票识别

#### 1.3 电子发票（PDF转PNG）
- **特点**: 清晰的电子格式、可能包含二维码
- **复杂度**: 中
- **用途**: 测试电子文档识别

#### 1.4 手写发票
- **特点**: 部分手写内容、混合打印和手写
- **复杂度**: 高
- **用途**: 测试手写识别能力

**图片要求**:
- 分辨率: 至少 1000x800 像素
- 格式: PNG 或 JPEG
- 内容: 包含完整的发票信息（发票号、日期、商品、金额等）

## 2. 📊 报表类 (Reports)

### 当前状态
- ⚠️ `sample_report.png` - 临时示例（建议替换）

### 建议收集的类型

#### 2.1 财务报表
- **资产负债表** (Balance Sheet)
  - 特点: 多列数据、资产/负债分类、总计行
  - 复杂度: 高
  - 用途: 测试复杂表格结构识别

- **利润表** (Income Statement)
  - 特点: 收入/支出分类、计算关系、百分比
  - 复杂度: 高
  - 用途: 测试数字和公式识别

- **现金流量表** (Cash Flow Statement)
  - 特点: 时间序列数据、分类汇总
  - 复杂度: 高
  - 用途: 测试时间序列数据提取

#### 2.2 数据报表
- **销售报表**: 包含商品、数量、金额、日期等
- **统计报表**: 包含图表、统计数据、百分比
- **财务报表**: 包含多维度数据分析

**图片要求**:
- 分辨率: 至少 1200x1600 像素（A4尺寸）
- 格式: PNG 或 JPEG
- 内容: 包含表格、数字、文字说明
- 特点: 清晰、对比度好、表格结构完整

## 📐 图片质量标准

### 通用要求

1. **分辨率**
   - 最小: 600x800 像素
   - 推荐: 1000x1400 像素以上
   - 复杂场景: 1200x1600 像素以上

2. **格式**
   - PNG（推荐，无损）
   - JPEG（可接受，质量85%以上）

3. **清晰度**
   - ✅ 文字清晰可读
   - ✅ 对比度良好
   - ✅ 无模糊、无重影
   - ❌ 避免过度压缩

4. **内容完整性**
   - ✅ 包含完整的文档信息
   - ✅ 无遮挡、无缺失
   - ✅ 表格结构完整

5. **文件大小**
   - 推荐: 100KB - 5MB
   - 最大: 10MB

---

## 🎯 复杂度分级

### 低复杂度
- 简单布局
- 清晰打印
- 标准格式
- **推荐数量**: 2-3个/类

### 中复杂度
- 多行多列表格
- 混合内容（文字+数字）
- 标准格式但有细节
- **推荐数量**: 3-5个/类

### 高复杂度
- 复杂表格结构
- 合并单元格
- 手写内容
- 多语言混合
- **推荐数量**: 2-3个/类

## 📊 收集建议

### 每类最少数量
- **发票**: 3-5个（不同类型）
- **报表**: 3-5个（不同报表类型）
- **表单**: 2-4个（不同表单类型）
- **收据**: 3-5个（不同场景）

### 优先级
1. **高优先级**: 发票（增值税发票）、财务报表（资产负债表）
2. **中优先级**: 表单（申请表）、收据（POS小票）
3. **低优先级**: 手写文档、多语言文档

## 🔄 使用流程

1. **收集图片** → 按照上述要求收集各类示例
2. **放置文件** → 放入对应的分类目录
3. **生成结果** → 运行OCR生成识别结果
4. **更新文档** → 更新README和索引文件

**生成识别结果命令**:
```bash
deepseek-ocr-cli \
  --prompt "<image>\n<|grounding|>Extract all text from this document." \
  --image assets/examples/[category]/[filename].png \
  --max-new-tokens 2048 \
  2>/dev/null > assets/examples/[category]/[filename]_result.txt
```
